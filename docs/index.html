Set up in 5 minutes

No Node.js. No server. No account required.

1

Download the ZIP

Go to the Releases page and download consentkit-ftp-vX.X.X.zip. Unzip it — you'll get four files.

2

Edit consentkit.config.json

Open it in any text editor and change these fields to match your site:

"title": "We use cookies",
"description": "We use cookies to improve your experience...",
"privacyPolicyUrl": "/privacy-policy",
"accentColor": "#4f46e5"  ← your brand colour
3

Upload to your server

Create a /consentkit/ folder on your web server and upload the files via FTP, cPanel File Manager, or SFTP:

yourwebsite.com/
└── consentkit/
    ├── widget.js
    ├── consentkit.config.json
    └── consent-log.php   ← optional logging
4

Paste one script tag

Add this before the </head> tag on every page. Remove data-log-url if you don't need server-side logging.

<script
  src="/consentkit/widget.js"
  data-config="/consentkit/consentkit.config.json"
  data-log-url="/consentkit/consent-log.php"
  defer
></script>

Reload your page — the consent banner will appear.

Block scripts until consent

Replace src with data-ck-src and add a data-ck-category attribute. ConsentKit holds the script back until the visitor accepts that category.

<!-- Google Analytics -->
<script
  data-ck-category="analytics"
  data-ck-src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXX"
  async
></script>

<!-- Facebook Pixel -->
<script
  data-ck-category="marketing"
  data-ck-src="https://connect.facebook.net/en_US/fbevents.js"
  async
></script>

<!-- YouTube embed -->
<iframe
  data-ck-category="functional"
  data-ck-src="https://www.youtube.com/embed/VIDEO_ID"
  width="560" height="315"
></iframe>

Config reference

Edit consentkit.config.json — no code required.

Field Type Description
versionstringBump to re-ask consent from returning visitors
banner.positionstringbottom-bar · bottom-left · bottom-right · center-popup
banner.primaryColorstringBanner background colour (e.g. #1a1a1a)
banner.accentColorstringButton & toggle colour — use your brand colour
banner.privacyPolicyUrlstringLink shown in banner description
banner.logoUrlstringOptional logo image displayed in the banner header
categories[].keystringUsed in data-ck-category attributes
categories[].lockedbooleantrue = always on (use for necessary)
categories[].defaultEnabledbooleanPre-selected state — set false for GDPR opt-in
consentLogging.enabledbooleanPOST consent records to consent-log.php
consentLogging.retentionDaysnumberAuto-delete old records (default: 1825 = 5 years)
googleConsentMode.enabledbooleanFire Google Consent Mode v2 signals
ccpa.enabledbooleanShow "Do Not Sell" link in banner footer

Compliance

Built-in protections — not configurable, not bypassable.

GDPR / ePrivacy

Opt-in only. All non-necessary scripts blocked before rendering. Accept and Reject buttons always identical in size and weight.

CCPA / CPRA

"Do Not Sell or Share My Personal Information" link triggers reject-all for non-necessary categories.

Google Consent Mode v2

All six parameters fired as denied before any user interaction, then updated on choice.

Global Privacy Control

When GPC is detected (Brave, Firefox + uBlock), the banner is skipped and all non-necessary categories are silently rejected.
